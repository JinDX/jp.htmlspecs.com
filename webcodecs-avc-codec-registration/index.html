概要

この登録は、[webcodecs-codec-registry] に記載される。 これは、AVC (H.264) について、(1) 完全修飾されたコーデック文字列、 (2) コーデック固有の EncodedVideoChunk [[internal data]] バイト、(3) VideoDecoderConfig.description バイト、 (4) EncodedVideoChunk [[type]] の値、 (5) VideoEncoderConfig へのコーデック固有の拡張、 および (6) VideoEncoderEncodeOptions へのコーデック固有の拡張を記述する。

この登録は、コーデック形式が知的財産権の主張によって制約されているかどうかに 関する情報を含めることを意図したものではない。実装者および著者が特定の コーデック形式を実装または使用する意図がある場合、この件について適切な 法律顧問に相談することが推奨される。WebCodecs の実装者は、AVC / H.264 コーデックを サポートする必要はない。

この登録は非規範的である。

1. 完全修飾コーデック文字列

コーデック文字列は接頭辞 "avc1." または "avc3." で始まり、それぞれ [rfc6381] の第3.4節および [iso14496-15] の第5.4.1節で説明される 6文字の接尾辞が続く。

2. EncodedVideoChunk データ

EncodedVideoChunk [[internal data]] は、[ITU-T-REC-H.264] の第7.4.1.2節で定義される アクセスユニットであることが期待される。

注: アクセスユニットは、ちょうど1つのプライマリ符号化 ピクチャを含む。

ビットストリームが avc 形式の場合、 [[internal data]] は、[iso14496-15] の第5.3.2節で定義される標準形式であると見なされる。

ビットストリームが annexb 形式の場合、 [[internal data]] は、[ITU-T-REC-H.264] Annex B で定義される Annex B 形式であると見なされる。

注: [[internal data]] は本質的にバイト境界に揃っているため、 実装はバイト境界の復元を要求されない。

3. VideoDecoderConfig description

description が存在する場合、それは [iso14496-15] の 第5.3.3.1節で定義される AVCDecoderConfigurationRecord であると見なされ、ビットストリームは avc 形式であると見なされる。

注: この形式は .MP4 ファイルで一般的に使用され、そこでは プレーヤーは通常、 メディアデータにランダムアクセスできる。

description が存在しない場合、ビットストリームは annexb 形式であると見なされる。

注: "annexb" 形式は、 [ITU-T-REC-H.264] の Annex B でより詳しく説明されている。この形式はライブストリーミングアプリケーションで一般的に使用され、 SPS および PPS データを定期的に含めることで、ユーザーがストリームの途中から 容易に開始できるようにする。

4. EncodedVideoChunk type

EncodedVideoChunk の [[type]] が key であり、ビットストリームが avc 形式である場合、その EncodedVideoChunk は、瞬時復号更新（IDR）ピクチャであるプライマリ符号化ピクチャを 含むことが期待される。

注: ビットストリームが avc 形式である場合、復号に必要なパラメータセットは VideoDecoderConfig.description に含まれる。

EncodedVideoChunk の [[type]] が key であり、ビットストリームが annexb 形式である場合、その EncodedVideoChunk は、瞬時復号更新（IDR）ピクチャであるプライマリ符号化ピクチャと、 その EncodedVideoChunk 内のすべてのビデオデータ NAL ユニットを復号するために必要なすべてのパラメータセットの両方を含むことが期待される。

5. VideoEncoderConfig 拡張

partial dictionary VideoEncoderConfig {
  AvcEncoderConfig avc;
};
avc, 型は AvcEncoderConfig
AVC (H.264) コーデック用のコーデック固有の構成オプションを含む。

5.1. AvcEncoderConfig

dictionary AvcEncoderConfig {
  AvcBitstreamFormat format = "avc";
};
format, 型は AvcBitstreamFormat、既定値は "avc"
出力 EncodedVideoChunk の形式を構成する。 AvcBitstreamFormat を参照。

5.2. AvcBitstreamFormat

enum AvcBitstreamFormat {
  "annexb",
  "avc",
};

AvcBitstreamFormat は、AVC パラメータセットの場所、および ビットストリームをパッケージ化する仕組みを決定する。

annexb
SPS および PPS データは、ビットストリーム全体にわたって定期的に含まれる。

注: この形式は [ITU-T-REC-H.264] の Annex B でより詳しく説明されている。この形式はライブストリーミングアプリケーションで一般的に使用され、 SPS および PPS データを定期的に含めることで、ユーザーがストリームの途中から 容易に開始できるようにする。

avc
SPS および PPS データはビットストリームに含まれず、代わりに [[output callback]] を通じて VideoDecoderConfig.description として、つまり EncodedVideoChunkMetadata.decoderConfig のものとして出力される。

注: この形式は [iso14496-15] の第5.3節でより詳しく説明されている。この形式は .MP4 ファイルで一般的に使用され、そこでは プレーヤーは通常メディアデータにランダムアクセスできる。

6. VideoEncoderEncodeOptions 拡張

partial dictionary VideoEncoderEncodeOptions {
  VideoEncoderEncodeOptionsForAvc avc;
};
avc, 型は VideoEncoderEncodeOptionsForAvc
[ITU-T-REC-H.264] コーデック用のコーデック固有のエンコードオプションを含む。

6.1. VideoEncoderEncodeOptionsForAvc

dictionary VideoEncoderEncodeOptionsForAvc {
  unsigned short? quantizer;
};
quantizer, 型は unsigned short、nullable
フレームごとの量子化値を設定する。 [ITU-T-REC-H.264] では、 量子化しきい値は 0 から 51 まで変化させることができる。
