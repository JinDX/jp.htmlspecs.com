概要

Observable APIは、非同期のイベントストリームを扱うための構成可能で使いやすい方法を提供します。

2. コアインフラストラクチャ

2.1. Subscriber インターフェイス

[Exposed=*]
interface Subscriber {
  undefined next(any value);
  undefined error(any error);
  undefined complete();
  undefined addTeardown(VoidFunction teardown);

  // Subscriberが作成されてから、complete()/error()が呼び出されるか、購読解除されるまでtrue。
  // complete()/error()内ではこの属性はtrue。
  readonly attribute boolean active;

  readonly attribute AbortSignal signal;
};

Subscriber は、順序付き集合である内部オブザーバを持ち、初期状態は空です。

Subscriber は、ティアダウンコールバックを持ち、これはVoidFunctionのリストで、初期状態は空です。

Subscriber は、購読コントローラを持ち、これはAbortControllerです。

Subscriber は、activeというbooleanを持ち、初期値はtrueです。

注: これは、Subscriberが購読を終了した後、自身が所有するコールバックを二度と呼び出さないことを保証するための管理用変数です。

activeゲッターの手順は、thisのactive booleanを返します。

signalゲッターの手順は、thisの購読コントローラのsignalを返します。

next(value) メソッドの手順:

  1. thisのactiveがfalseなら、returnする。

  2. thisの関連グローバルオブジェクトがWindow オブジェクトであり、その関連付けられたDocumentが完全にアクティブでない場合、returnする。

  3. thisの内部オブザーバの各observerについて:

    1. observerのnext手順をvalueで実行する。

      保証: 例外は投げられなかった。

      注: ここで例外が投げられない理由は、内部オブザーバのnext手順がスクリプトによるコールバックのラッパーである場合、observerの処理手順が例外をキャッチしてグローバルへ報告するためです。

      next手順が仕様アルゴリズムである場合、手順自体で外部に例外を投げないようにしています。

error(error) メソッドの手順:

  1. thisのactiveがfalseなら、例外を報告し、errorとthisの関連グローバルオブジェクトで、returnする。

  2. thisの関連グローバルオブジェクトがWindow オブジェクトであり、その関連付けられたDocumentが完全にアクティブでない場合、returnする。

  3. 購読を終了 this。

  4. thisの内部オブザーバの各observerについて:

    1. observerのerror手順をerrorで実行する。

      保証: 例外は投げられなかった。

      注: 詳細はnext()のドキュメントを参照してください。

complete() メソッドの手順:

  1. thisのactiveがfalseなら、returnする。

  2. thisの関連グローバルオブジェクトがWindow オブジェクトであり、その関連付けられたDocumentが完全にアクティブでない場合、returnする。

  3. 購読を終了 this。

  4. thisの内部オブザーバの各observerについて:

    1. observerのcomplete手順を実行する。

      保証: 例外は投げられなかった。

      注: 詳細はnext()のドキュメントを参照してください。

addTeardown(teardown) メソッドの手順:

  1. thisの関連グローバルオブジェクトがWindow オブジェクトであり、その関連付けられたDocumentが完全にアクティブでない場合、returnする。

  2. thisのactiveがtrueなら、teardownをthisのティアダウンコールバックリストに追加する。

  3. そうでなければ、teardownを«»と"report"で呼び出す。

購読を終了するには、Subscriber subscriberと、オプションのany reasonを与えて、以下の手順を実行する:

  1. subscriberのactiveがfalseなら、returnする。

    これは再入呼び出しを防ぐためです。たとえば「プロデューサ主導の購読解除」ケース:

    const outerController = new AbortController();
const observable = new Observable(subscriber => {
  subscriber.addTeardown(() => {
    // 2.) このteardownは実行中の「Close」アルゴリズム内部で実行されます。
    //     下流のsignalをabortするとそのabortアルゴリズムが実行され、その一つが今まさに実行中の「Close」アルゴリズムです。
    outerController.abort();
  });

  // 1.) これは即座に「Close」アルゴリズムを呼び出し、subscriber.activeをfalseに設定します。
  subscriber.complete();
});

observable.subscribe({}, {signal: outerController.signal});

  2. subscriberのactive booleanをfalseにする。

  3. Signal abort subscriberの購読コントローラにreasonがあれば渡す。

  4. subscriberのティアダウンコールバックを逆順で各teardownについて:

    1. subscriberの関連グローバルオブジェクトがDocument オブジェクトで、その関連付けられたDocumentが完全にアクティブでないなら、これ以降の手順を中止する。

      注: この手順は各teardownごとに繰り返し実行される可能性があります。なぜなら各ティアダウンでDocumentが非アクティブになる場合があるためです。

    2. teardownを«»と"report"で呼び出す。

2.2. Observable インターフェイス

// SubscribeCallbackはObservable「作成者」のコードが存在する場所です。
// subscribe()が呼び出された時に呼び出され、新しい購読をセットアップします。
callback SubscribeCallback = undefined (Subscriber subscriber);
callback ObservableSubscriptionCallback = undefined (any value);

dictionary SubscriptionObserver {
  ObservableSubscriptionCallback next;
  ObservableSubscriptionCallback error;
  VoidFunction complete;
};

callback ObservableInspectorAbortHandler = undefined (any value);

dictionary ObservableInspector {
  ObservableSubscriptionCallback next;
  ObservableSubscriptionCallback error;
  VoidFunction complete;

  VoidFunction subscribe;
  ObservableInspectorAbortHandler abort;
};

typedef (ObservableSubscriptionCallback or SubscriptionObserver) ObserverUnion;
typedef (ObservableSubscriptionCallback or ObservableInspector) ObservableInspectorUnion;

dictionary SubscribeOptions {
  AbortSignal signal;
};

callback Predicate = boolean (any value, unsigned long long index);
callback Reducer = any (any accumulator, any currentValue, unsigned long long index);
callback Mapper = any (any value, unsigned long long index);
// Mapperと異なるのは戻り値の型のみで、このコールバックはシーケンス内の各要素を訪問するためだけに使われ、変換はしません。
callback Visitor = undefined (any value, unsigned long long index);

// このコールバックは`any`を返し、その値は`Observable`へ変換される必要があります（`Observable`変換セマンティクスによる）。
callback CatchCallback = any (any value);

[Exposed=*]
interface Observable {
  constructor(SubscribeCallback callback);
  undefined subscribe(optional ObserverUnion observer = {}, optional SubscribeOptions options = {});

  // valueが以下のいずれかの場合、ネイティブObservableを構築します：
  //   - Observable
  //   - AsyncIterable
  //   - Iterable
  //   - Promise
  static Observable from(any value);

  // Observableを返すオペレーター。詳しくは仕様の「オペレーター」節を参照。
  //
  // takeUntil()はPromise、Iterable、AsyncIterable、他のObservableを消費できます。
  Observable takeUntil(any value);
  Observable map(Mapper mapper);
  Observable filter(Predicate predicate);
  Observable take(unsigned long long amount);
  Observable drop(unsigned long long amount);
  Observable flatMap(Mapper mapper);
  Observable switchMap(Mapper mapper);
  Observable inspect(optional ObservableInspectorUnion inspectorUnion = {});
  Observable catch(CatchCallback callback);
  Observable finally(VoidFunction callback);

  // Promiseを返すオペレーター。
  Promise<sequence<any>> toArray(optional SubscribeOptions options = {});
  Promise<undefined> forEach(Visitor callback, optional SubscribeOptions options = {});
  Promise<boolean> every(Predicate predicate, optional SubscribeOptions options = {});
  Promise<any> first(optional SubscribeOptions options = {});
  Promise<any> last(optional SubscribeOptions options = {});
  Promise<any> find(Predicate predicate, optional SubscribeOptions options = {});
  Promise<boolean> some(Predicate predicate, optional SubscribeOptions options = {});
  Promise<any> reduce(Reducer reducer, optional any initialValue, optional SubscribeOptions options = {});
};

Observable は、subscribe callbackを持ちます。これはSubscribeCallbackまたはSubscriberを受け取る一連の手順です。

Observable は、weak subscriberを持ちます。これはSubscriberへの弱参照（またはnull）で、初期値はnullです。

注: これら型の「union」は、JavaScriptによって作成されたObservable（必ずSubscribeCallback付きで構築される）と、ネイティブで構築されたObservable（subscribe callbackがJavaScriptコールバックでなく任意のネイティブ手順の場合）双方をサポートするためです。when()の戻り値が後者の例です。

new Observable(callback) コンストラクターの手順:

  1. thisのsubscribe callbackをcallbackに設定する。

    注: このコールバックは後でsubscribe()が呼ばれた時に実行されます。

subscribe(observer, options) メソッドの手順:

  1. thisにobserverとoptionsで購読する。

2.2.1. 補助概念

デフォルトエラーアルゴリズムは、any 型のerrorを受け取るアルゴリズムで、次の手順を実行します：

  1. 例外を報告する、引数はerrorと、current realm（現行レルム）のグローバルオブジェクト。

注: このデフォルト手順を分離することで、仕様文書内でネイティブにObservable へ購読する（すなわち、仕様プローズから購読し、subscribe() メソッドを経由しない場合）たびに、この手順を繰り返し定義する必要がなくなります。

内部オブザーバは、以下の構造体を持つ項目を持ちます：

next手順

型anyの単一パラメータを受け取るアルゴリズム。初期状態では何もしません。

error手順

型anyの単一パラメータを受け取るアルゴリズム。初期状態ではデフォルトエラーアルゴリズム。

complete手順

パラメータなしのアルゴリズム。初期状態では何もしません。

内部オブザーバ構造体は、next、 error、 complete のコールバック関数を反映するために使われます。 JavaScriptからObservable に対してsubscribe()メソッドで購読する場合、 これらのアルゴリズム「手順」は、スクリプトが提供したnext、 error、 complete のコールバック関数を呼び出すラッパーとなります。

しかし、仕様プローズ（ユーザスクリプトではない）がObservable に購読する場合は、これらの「手順」はWeb IDLのコールバック関数入りのObserverUnionで渡されるのではなく、任意の仕様アルゴリズムになります。例えば、§ 2.3.3 Promiseを返すオペレーターなどで利用されます。

任意のany型valueをObservableに変換するには、次の手順を実行する：

注: このアルゴリズムはWeb IDLのfrom()メソッドから分離しており、仕様本文からWeb IDLバインディングを経由せずに値をObservableに変換できるようにしている。

  1. Type(value)がObjectでない場合、TypeErrorをTypeErrorで投げる。

    注: これによりプリミティブ型（例：文字列）がイテラブルとして変換されるのを防ぎます。詳細はWICG/observable#125を参照。

  2. Observableから: valueの具体型がObservableなら、valueを返す。

  3. 非同期イテラブルから: asyncIteratorMethodを? GetMethod(value, %Symbol.asyncIterator%)とする。

    注: async iteratorプロトコルのチェックにはGetMethodを利用します。GetIteratorよりも、未実装時に例外を投げません。GetIteratorは、プロトコルが未実装または呼び出し不可・getterが例外を投げる場合、両方で例外となります。GetMethodは後者のみ例外。

  4. asyncIteratorMethodがundefinedまたはnullなら、イテラブルからにジャンプ。

  5. nextAlgorithmを以下の手順とする：引数はSubscriber型subscriberと、Iterator Record型iteratorRecord。

    1. subscriberの購読コントローラのsignalがabortedならreturn。

    2. nextPromiseをPromiseまたはundefined（初期値undefined）とする。

    3. nextCompletionをIteratorNext(iteratorRecord)で求める。

      注: ここではnext()がPromise/thenableを返す場合に備え、IteratorStepValueではなくIteratorNextを使います。

    4. nextCompletionがthrow completionなら：

      1. 保証：iteratorRecordの[[Done]]はtrue。

      2. nextPromiseを拒否されたPromise（nextRecordの[[Value]]）とする。

    5. それ以外でnextRecordがnormal completionなら、nextPromiseを解決されたPromise（nextRecordの[[Value]]）とする。

      注: nextRecordの[[Value]]自体がまだPromiseでない場合もあるため。

    6. nextPromiseがsettleしたらリアクションする：

      • nextPromiseがiteratorResultでfulfilledされた場合：

        1. Type(iteratorResult) がObjectでなければ、subscriberのerror() メソッドをTypeErrorで呼び出し、以降中止。

        2. doneをIteratorComplete(iteratorResult)とする。

        3. doneがthrow completionなら、subscriberのerror() メソッドをdoneの[[Value]]で呼び出し、以降中止。

        4. doneの[[Value]]がtrueなら、subscriberのcomplete() を呼び出し、以降中止。

        5. valueをIteratorValue(iteratorResult)とする。

        6. valueがthrow completionなら、subscriberのerror() メソッドをvalueの[[Value]]で呼び出し、以降中止。

        7. subscriberのnext() をvalueの[[Value]]で呼び出す。

        8. nextAlgorithmをsubscriberとiteratorRecordで再実行。

      • nextPromiseがrでrejectedされた場合、subscriberのerror()にrを渡す。

  6. 新しいObservable を返す。そのsubscribe callbackはSubscriber型subscriberを受け取って以下を行う：

    1. subscriberの購読コントローラのsignalがabortedならreturn。

    2. iteratorRecordCompletionをGetIterator(value, async)とする。

      注: これによりgetterを再度呼び出しプロトコルを実行するため、テスト期待値に合わせています。詳細はissue#127参照。

    3. iteratorRecordCompletionがthrow completionなら、subscriberのerror()メソッドでiteratorRecordCompletionの[[Value]]を渡し、この手順を中止。

      注: この場合は購読に対して同期的にerror()が呼ばれます。通常はPromiseのrejectとして非同期でobserverへ伝播します。

    4. iteratorRecordを!iteratorRecordCompletionとする。

    5. 保証：iteratorRecordはIterator Record。

    6. subscriberの購読コントローラのsignalがabortedならreturn。

    7. 次のabortアルゴリズムをsubscriberの購読コントローラのsignalへ追加：

      1. AsyncIteratorClose(iteratorRecord, NormalCompletion(subscriberの購読コントローラのabort理由))を実行。

    8. nextAlgorithmをsubscriberとiteratorRecordで実行。

  7. イテラブルから: iteratorMethodを? GetMethod(value, %Symbol.iterator%)とする。

  8. iteratorMethodがundefinedなら、Promiseからにジャンプ。そうでなければ、新しい Observable を返す。そのsubscribe callbackはSubscriber型subscriberを受け取って以下を実行：

    1. subscriberの購読コントローラのsignalがabortedならreturn。

    2. iteratorRecordCompletionをGetIterator(value, sync)とする。

    3. iteratorRecordCompletionがthrow completionなら、subscriberのerror()メソッドでiteratorRecordCompletionの[[Value]]を渡し、この手順を中止。

    4. iteratorRecordを!iteratorRecordCompletionとする。

    5. subscriberの購読コントローラのsignalがabortedならreturn。

    6. 次のabortアルゴリズムをsubscriberの購読コントローラのsignalへ追加：

      1. IteratorClose(iteratorRecord, NormalCompletion(UNUSED))を実行。

    7. while true:

      1. nextをIteratorStepValue(iteratorRecord)とする。

      2. nextがthrow completionなら、subscriberのerror()メソッドでnextの[[Value]]を渡し、breakする。

      3. nextを!で評価。

      4. もしnextがdoneなら：

        1. 保証：iteratorRecordの[[Done]]はtrue。

        2. subscriberのcomplete()を呼ぶ。

        3. return。

      5. subscriberのnext()にnextを渡して呼ぶ。

      6. subscriberの購読コントローラのsignalがabortedならbreakする。

  9. Promiseから: IsPromise(value)がtrueなら：

    1. 新しい Observable を返す。そのsubscribe callbackはSubscriber型subscriberを受け取って以下を行う：

      1. valueに反応：

        1. valueがvでfulfilledされた場合：

          1. subscriberのnext()にvを渡して呼ぶ。

          2. subscriberのcomplete()を呼ぶ。

        2. valueがrでrejectedされた場合、subscriberのerror()にrを渡して呼ぶ。

  10. TypeErrorを投げる。

      3. observerのcompleteが存在する場合、internal observerのcomplete手順を、以下を実行するステップに設定する：

        1. observerのcompleteを«»と"report"で呼び出す。

      もしobserverが内部オブザーバ型の場合
      internal observerをobserverに設定する。

  4. 保証：internal observerのerror手順はデフォルトエラーアルゴリズムまたは、与えられたerrorのコールバック関数を呼び出すアルゴリズムであること。

  5. thisのweak subscriberがnullでなく、かつthisのweak subscriberのactiveがtrueなら：

    1. subscriberをthisのweak subscriberとする。

    2. internal observerをsubscriberの内部オブザーバに追加する。

    3. optionsのsignalが存在する場合：

      1. optionsのsignalがabortedなら、internal observerをsubscriberの内部オブザーバから削除する。

      2. それ以外の場合、次のabortアルゴリズムをoptionsのsignalへ追加：

        1. subscriberのactiveがfalseなら以降中止。

        2. internal observerをsubscriberの内部オブザーバから削除する。

        3. subscriberの内部オブザーバが空なら、subscriberを終了する（abort理由はoptionsのsignalのabort理由）。

    4. return。

  6. subscriberを新しいSubscriberとする。

  7. internal observerをsubscriberの内部オブザーバに追加する。

  8. thisのweak subscriberをsubscriberに設定する。

  9. optionsのsignalが存在する場合：

    1. optionsのsignalがabortedなら、subscriberを終了する（abort理由はoptionsのsignalのabort理由）。

    2. それ以外の場合、次のabortアルゴリズムをoptionsのsignalへ追加：

      1. subscriberのactiveがfalseなら以降中止。

      2. internal observerをsubscriberの内部オブザーバから削除する。

      3. subscriberの内部オブザーバが空なら、subscriberを終了する（abort理由はoptionsのsignalのabort理由）。

  10. thisのsubscribe callbackがSubscribeCallback型の場合、«subscriber»と"rethrow"で呼び出す。

    もし例外Eが投げられたら、subscriberのerror()にEを渡す。

  11. そうでなければ、thisのsubscribe callbackの手順にsubscriberを渡して実行する。

2.3.2. Observableを返す オペレーター

takeUntil(value) メソッドの手順:

  1. sourceObservableをthisとする。

  2. notifierを、valueをObservableへ変換した結果とする。

  3. observableを新しい Observable とし、そのsubscribe callbackは、 Subscriber型subscriberを受け取り以下を実行するアルゴリズムとする:

    このメソッドは2つのObservableに対して購読を行います: (1) notifier、(2) sourceObservable。以下の状況で両方の購読を解除します:

    1. notifierが値（"next"または"error"）を出し始めた場合。 この場合、必要な情報が得られたためnotifierの購読を解除し、sourceObservableの値をこのメソッドのobservableへ流す必要がなくなるため、observableの購読も終了する（= sourceObservableの購読も解除）。
