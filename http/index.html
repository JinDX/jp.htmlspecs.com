📄 HTTP ドキュメント

HTTP はこれらの IETF の RFCs および IANA の レジストリ によって定義されています。

コア仕様

HTTP プロトコルの「コア」セマンティクスは次のもので定義されています：

  • RFC 9110: HTTP Semantics HTTP セマンティクス
  • RFC 9111: HTTP Caching HTTP キャッシング

それらのセマンティクスは「on the wire」で3つの方法で表現されます：

  • RFC 9112: HTTP/1.1 HTTP/1.1
  • RFC 9113: HTTP/2 HTTP/2
  • RFC 9114: HTTP/3 HTTP/3

HTTP の後続バージョンではフィールド圧縮が提供されています：

  • RFC 7541: HPACK Header Compression for HTTP/2 HPACK ヘッダ圧縮 (HTTP/2)
  • RFC 9204: QPACK Field Compression for HTTP/3 QPACK フィールド圧縮 (HTTP/3)

これらの RFC は、HTTP を定義していたそれ以前のすべての RFC をまとめて廃止します。これには RFC 1945, RFC 2068, RFC 2616, RFC 2617, RFC 7230-5, および RFC 7540 が含まれます。

また、増えつつある多くのヘッダやトレーラで使用されている Structured Field Values 仕様も関連しています：

  • RFC 9651: Structured Field Values for HTTP (test suite) HTTP の構造化フィールド値

HTTP の「上に」構築されたプロトコルは、ここに文書化されているベスト・カレント・プラクティスに従うべきです：

  • RFC 9205: Building Protocols with HTTP HTTP でプロトコルを構築する

拡張レジストリ

メソッド、ステータスコード、ヘッダを含む多数の HTTP 拡張があります。適切なレジストリに登録されている限り、それらはすべて HTTP の「一部」であることに注意してください：

  • HTTP Method Registry - 詳細
  • HTTP Status Code Registry - 詳細
  • HTTP Field Name Registry - 詳細
  • HTTP/2 Parameters - フレームタイプ、設定、エラーコード
  • HTTP/3 Parameters - フレームタイプ、設定、エラーコード

個々のヘッダには許可される値を規定する独自のレジストリがあり、次のものが含まれます：

  • HTTP Authentication Scheme Registry - 詳細
  • HTTP Cache Directive Registry - 詳細
  • HTTP Parameters - Content-codings、transfer-codings、Forwarded、Prefer、および Range 関連のヘッダ
