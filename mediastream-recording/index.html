1. 概要

この API は基本的な録画を非常に簡単に行えるようにすることを目指しつつ、より複雑なユースケースにも対応できるようにしています。最も単純な場合、アプリケーションは MediaRecorder オブジェクトをインスタンス化し、start() を呼び出し、その後 stop() を呼び出すか、MediaStreamTrack（複数可） が終了するのを待ちます [GETUSERMEDIA]。録画の内容はプラットフォームのデフォルトエンコーディングで ondataavailable イベントを通じて利用可能になります。プラットフォームが利用可能なエンコーディングの一覧を問い合わせたり、必要に応じて望ましいものを選択したりするための関数が提供されています。アプリケーションは一度に受け取りたいデータ量を選択することもできます。デフォルトでは、録画が終了したときに録画全体を含む Blob が返されます。ただし、アプリケーションは定期的により小さなデータバッファを受け取るように選択することもできます。

2. メディアレコーダー API

[Exposed=Window]
interface MediaRecorder : EventTarget {
  constructor(MediaStream stream, optional MediaRecorderOptions options = {});
  readonly attribute MediaStream stream;
  readonly attribute DOMString mimeType;
  readonly attribute RecordingState state;
  attribute EventHandler onstart;
  attribute EventHandler onstop;
  attribute EventHandler ondataavailable;
  attribute EventHandler onpause;
  attribute EventHandler onresume;
  attribute EventHandler onerror;
  readonly attribute unsigned long videoBitsPerSecond;
  readonly attribute unsigned long audioBitsPerSecond;
  readonly attribute BitrateMode audioBitrateMode;

  undefined start(optional unsigned long timeslice);
  undefined stop();
  undefined pause();
  undefined resume();
  undefined requestData();

  static boolean isTypeSupported(DOMString type);
};

2.1. コンストラクタ

MediaRecorder(MediaStream stream, optional MediaRecorderOptions options = {})

MediaRecorder() コンストラクタが呼び出されたとき、ユーザーエージェントは次の手順を実行しなければなりません（MUST）：

  1. コンストラクタの第一引数を stream とする。

  2. コンストラクタの第二引数を options とする。

  3. is type supported を options の mimeType メンバーを引数として呼び出した結果が false を返す場合は、NotSupportedError DOMException をスローしてこれらの手順を中止する。

  4. 新しく構築された MediaRecorder オブジェクトを recorder に割り当てる。

  5. recorder に [[ConstrainedMimeType]] 内部スロットを持たせ、これは options の mimeType メンバーの値で初期化する。

  6. recorder に [[ConstrainedBitsPerSecond]] 内部スロットを持たせ、これは options の bitsPerSecond メンバーが存在する場合はその値で、そうでなければ null で初期化する。

  7. recorder に [[VideoKeyFrameIntervalDuration]] 内部スロットを持たせ、これは options の videoKeyFrameIntervalDuration メンバーが存在する場合はその値で、そうでなければ null で初期化する。

  8. recorder に [[VideoKeyFrameIntervalCount]] 内部スロットを持たせ、これは options の videoKeyFrameIntervalCount メンバーが存在する場合はその値で、そうでなければ null で初期化する。

  9. recorder の stream 属性を stream に初期化する。

  10. recorder の mimeType 属性を recorder の [[ConstrainedMimeType]] スロットの値で初期化する。

  11. recorder の state 属性を inactive に初期化する。

  12. recorder の videoBitsPerSecond 属性を、options の videoBitsPerSecond メンバーの値で初期化する（存在する場合）。存在しない場合は、ユーザーエージェントが妥当と判断するビデオ用の目標値を選択する。

  13. recorder の audioBitsPerSecond 属性を、options の audioBitsPerSecond メンバーの値で初期化する（存在する場合）。存在しない場合は、ユーザーエージェントが妥当と判断するオーディオ用の目標値を選択する。

  14. もし recorder の [[ConstrainedBitsPerSecond]] スロットが null でない場合、recorder の videoBitsPerSecond および audioBitsPerSecond 属性を、それぞれのメディアタイプに対してユーザーエージェントが妥当とみなす値に設定し、videoBitsPerSecond と audioBitsPerSecond の合計が recorder の [[ConstrainedBitsPerSecond]] スロットの値に近くなるようにする。

  15. もし recorder が BitrateMode を、options の audioBitrateMode メンバーで指定された値でサポートしている場合、recorder の audioBitrateMode 属性を options の同名メンバーの値で初期化する。そうでない場合は recorder の audioBitrateMode 属性を "variable" に初期化する。

  16. recorder を返す。

2.2. 属性

stream, of type MediaStream, readonly

MediaStream [GETUSERMEDIA] を録画するために使用します。

mimeType, of type DOMString, readonly

MediaRecorder オブジェクトが使用する MIME タイプ（[RFC2046]）。

ユーザーエージェントは、録画に対応する任意の MIME タイプを再生できるべきです。例えば、録画されたビデオを HTML の <video> タグで再生できるべきです。

mimeType は type/subtype の組み合わせで録画のメディアタイプとコンテナ形式を指定し、不明瞭な場合は codecs や profiles パラメータを [RFC6381] に従って指定します。個々のコーデックはさらに任意の特定パラメータを持つ場合があります。

state, of type RecordingState, readonly

現在の MediaRecorder オブジェクトの状態。

onstart, of type EventHandler

start イベントを処理するために呼び出されます。

onstop, of type EventHandler

stop イベントを処理するために呼び出されます。

ondataavailable, of type EventHandler

dataavailable イベントを処理するために呼び出されます。録画データの Blob はこのイベントに含まれ、その data 属性からアクセスできます。

onpause, of type EventHandler

pause イベントを処理するために呼び出されます。

onresume, of type EventHandler

resume イベントを処理するために呼び出されます。

onerror, of type EventHandler

エラーを表す ErrorEvent を処理するために呼び出されます。

videoBitsPerSecond, of type unsigned long, readonly

ビデオトラックのエンコードに使用される目標ビットレート。

audioBitsPerSecond, of type unsigned long, readonly

オーディオトラックのエンコードに使用される目標ビットレート。

audioBitrateMode, of type BitrateMode, readonly

BitrateMode がオーディオトラックのエンコードに使用されます。

2.3. メソッド

歴史的理由により、以下のメソッドは state を同期的に変更し、イベントは非同期で発火されます。
start(optional unsigned long timeslice)
MediaRecorder オブジェクトの start() メソッドが呼び出されたとき、UAは次の手順を実行しなければなりません（MUST）:
  1. recorder を、このメソッドが呼ばれた MediaRecorder オブジェクトとする。
  2. timeslice をメソッドの第1引数（指定されていれば）、なければ undefined とする。
  3. stream を recorder の stream 属性の値とする。
  4. tracks を stream の track set 内の live なトラックの集合とする。
  5. recorder の state 属性の値が inactive でない場合は、InvalidStateError DOMException をスローし、この手順を中断する。
  6. stream の isolation properties が recorder からのアクセスを許可しない場合、 SecurityError DOMException をスローし、中断する。
  7. stream が inactive の場合、NotSupportedError DOMException をスローし、中断する。
  8. [[ConstrainedMimeType]] スロットでメディアタイプやコンテナ、コーデックが指定されていれば、recorder の設定をその値に制約する。
  9. recorder の [[ConstrainedBitsPerSecond]] スロットが null 以外の場合、 tracks のすべての録音用に、ユーザーエージェントが妥当とみなす値で recorder の videoBitsPerSecond および audioBitsPerSecond 属性値を設定し、それらの合計が recorder の [[ConstrainedBitsPerSecond]] スロット値に近くなるようにする。
  10. videoBitrate を recorder の videoBitsPerSecond 属性値とし、録画するすべてのビデオトラックに対してビットレート制約（目安）を与える。videoBitrate はエンコーダへのヒントであり、必ずしも正確に達成されるとは限らない。
  11. audioBitrate を recorder の audioBitsPerSecond 属性値とし、録画するすべてのオーディオトラックに対してビットレート制約（目安）を与える。audioBitrate も同様にヒント扱いとなる。
  12. videoKeyFrameIntervalDuration を recorder.[[VideoKeyFrameIntervalDuration]] とし、 videoKeyFrameIntervalCount を recorder.[[VideoKeyFrameIntervalCount]] とする。UAは 下記のルールでビデオエンコーダの動作を制約するべき（SHOULD）:
    • videoKeyFrameIntervalDuration が null でなく、かつ videoKeyFrameIntervalCount が null の場合： 最後のキーフレームから videoKeyFrameIntervalDuration ミリ秒経過後の最初のフレームでキーフレームを出力する。
    • videoKeyFrameIntervalCount が null でなく、かつ videoKeyFrameIntervalDuration が null の場合： 最後のキーフレームから videoKeyFrameIntervalCount 枚後の最初のフレームでキーフレームを出力する。
    • どちらも null でない場合： NotSupportedError DOMException をスローし、この処理を中止する。
    • いずれも null の場合： UAは適切にキーフレームを出力してよい。

    なお、エンコーダはキーフレーム出力のタイミングを独自に判断することがあります。

  13. recorder の全オーディオトラックについては BitrateMode 指定値でエンコードするよう構成を制約する。
  14. tracks 内の各トラックについて、UAが現在の設定で録音不可なら NotSupportedError DOMException をスローし、中止する。
  15. recorder の state を recording にセットし、次を並列で実行する:
    1. 収録フォーマット・コーデックがまだ完全に決定していない場合は、UAがその設定を決定する。トラックのソースにより何を使うか考慮してもよい（MAY）。
      開始時点で tracks のソースをみて、再エンコードを避けるための構成をUAが選ぶこともできる。例えば MediaStreamTrack が RTCPeerConnection 由来のリモートトラックならRTPストリームのコーデックに合わせるなど。ただし録画途中でRTP側のコーデック変更が発生した場合UAはそれを再エンコードできる必要がある。
    2. recorder の現設定で tracks の録画を開始しデータを Blob blob として収集する。次をDOMマニピュレーションタスクソースでキューイングする:
      1. extendedMimeType を recorder の [[ConstrainedMimeType]] 値とする。
      2. 必要に応じ MediaRecorder の設定を反映したmedia type/subtype, codecs, profilesパラメータ([RFC6381] や追加パラメータ)を extendedMimeType に付加する。
      3. recorder の mimeType 属性に extendedMimeType をセットする。
      4. start イベントを recorder に発火する。
    3. stream の isolation properties が途中からアクセス禁止に変わった場合は、即座にデータ収集を停止し、取得済みデータは破棄し、次をキューする:
      1. Inactivate the recorder を recorder で呼ぶ。
      2. SecurityError エラーイベントを recorder に発火。
      3. dataavailable Blobイベントを recorder（blob付き）で発火。
      4. stop イベントを recorder に発火。
    4. stream の track set にトラックが追加・削除された時は、即座に収集停止＆次をキュー:
      1. Inactivate the recorder を recorder で呼ぶ。
      2. InvalidModificationError エラーイベントを発火。
      3. dataavailable Blobイベントを recorder（blob付き）で発火。
      4. stop イベントを recorder に発火。
    5. UAが isolation properties や stream のtrack set以外の理由でデータ収集を続行できない場合も、即座に収集停止＆次をキュー:
      1. Inactivate the recorder を recorder で呼ぶ。
      2. UnknownError エラーイベントを発火。
      3. dataavailable Blobイベントを recorder（blob付き）で発火。
      4. stop イベントを recorder に発火。
    6. timeslice が undefined でなければ、最小でも timeslice ミリ秒分のデータ収集後（またはUA独自の最小値以上）、新たな Blob blob にデータ収集を行い、dataavailable Blobイベントをキューイング。

      timeslice に undefined を指定すると最大の unsigned long 値とみなします。

    7. 全トラックが ended になった場合、収集停止＆次のとおりキューイング:
      1. Inactivate the recorder を recorder で呼ぶ。
      2. dataavailable Blobイベントを recorder（blob付き）で発火。
      3. stop イベントを recorder に発火。

stop()、 requestData()、 pause() も録画処理に影響します。

UAは stream を元のTrackが再現可能なかたちで記録しなければならず（MUST）、 Blob が複数出力された場合 （timeslice や requestData() により）、 個別のBlobは単独で再生できなくてもよいが、録画終了時点での全てのBlobを連結したものは再生できなければならない（MUST）。

MediaStream 内のトラックのいずれかが muted または enabled でない場合、そのTrackが出力する内容どおり、UAは黒フレームや無音のみを記録します。

Inactivate the recorder アルゴリズム（引数に recorder）は次の通り:

  1. recorder の mimeType 属性に [[ConstrainedMimeType]] スロット値をセット。
  2. recorder の state 属性を inactive にセット。
  3. recorder の [[ConstrainedBitsPerSecond]] スロットが undefined でなければ、 各メディアタイプで妥当な videoBitsPerSecond および audioBitsPerSecond 属性に設定し、その合計が [[ConstrainedBitsPerSecond]] スロット値に近くなるようにする。
stop()
MediaRecorder オブジェクトの stop() メソッドが呼ばれたとき、UAは以下を実行すること:
  1. recorder を、このメソッドが呼ばれた MediaRecorder オブジェクトとする。
  2. recorder の state 属性が inactive なら、この手順を中断する。
  3. Inactivate the recorder を recorder で呼ぶ。
  4. DOMマニピュレーションタスクソースで以下をキューする:
    1. データ収集を停止。
    2. 収集済みデータを blob とし、dataavailable Blobイベント（recorderとblob）を発火。
    3. stop イベントを recorder に発火。
  5. undefined を返す。
pause()
MediaRecorder オブジェクトの pause() メソッド呼出時、UAは以下の手順を実施する:
  1. state が inactive なら InvalidStateError DOMException をスローし、終了。
  2. state が paused なら中断。
  3. state を paused にし、以下をキューイング:
    1. 以降のデータは blob へ収集せず（録音再開まで保持）。
    2. target を MediaRecorder 本体とし、pause イベントを target に発火。
  4. undefined を返す。
resume()
MediaRecorder オブジェクトの resume() メソッド呼出時、UAは次の手順を行う:
  1. state が inactive なら InvalidStateError DOMException をスローして終了。
  2. state が recording なら中断。
  3. state を recording にし、以下をキューイング:
    1. 現blobへのデータ収集を再開。
    2. target を MediaRecorder 本体とし、resume イベントを target に発火。
  4. undefined を返す。
requestData()
MediaRecorder オブジェクトの requestData() メソッド呼出時、UAは次の手順を行う:
  1. state が inactive なら InvalidStateError DOMException をスローし、終了。そうでなければ次内容のタスクをキューイング:
    1. これまで収集したデータを blob（Blob）として target に dataavailable Blobイベントを発火する。 （blob が空の場合もある）
    2. 新たなBlobを作成し、以降のデータをそちらに収集する。
  2. undefined を返す。
isTypeSupported(DOMString type)
指定した MIME タイプで MediaRecorder が記録可能かを調べます。 true が返されても、単にその MediaRecorder 実装が指定MIME型の Blob を記録できると示すだけであって、充分なリソースがあれば必ずしも成功するとは限りません。このメソッド呼出時、UAは is type supported アルゴリズムの結果を返すこと。
is type supported アルゴリズムは次の通り:
  1. type をアルゴリズムの引数とする。
  2. type が空文字列なら true を返す（この場合、コンテナとコーデック選択をUA任せにすることと実質等しい）。
  3. type が正しい MIME型文字列を含まなければ false。
  4. type が MediaRecorder 非対応のメディアタイプまたはサブタイプを含めば false。
  5. type がMediaRecorder非対応のコンテナを含めば false。
  6. type に2つ以上のオーディオコーデック/ビデオコーデックがあればfalse。
  7. type に非対応コーデックがあれば false。
  8. MediaRecorderが指定 type/subtype, codecs, containerの組合せをサポートしなければfalse。
  9. trueを返す。

2.4. データの取り扱い

blob イベントを発火する とは、Blob blob を BlobEvent の data 属性に初期化し、target で イベントを発火することを意味します。

通常 blob には UA が直近で recording state へ遷移した後に収集したデータが入ります。

2.5. MediaRecorderOptions

dictionary MediaRecorderOptions {
  DOMString mimeType = "";
  unsigned long audioBitsPerSecond;
  unsigned long videoBitsPerSecond;
  unsigned long bitsPerSecond;
  BitrateMode audioBitrateMode = "variable";
  DOMHighResTimeStamp videoKeyFrameIntervalDuration;
  unsigned long videoKeyFrameIntervalCount;
};

2.5.1. メンバー

mimeType, 型 DOMString、デフォルト値 ""
録画のコンテナおよびコーデック形式 [RFC2046]。形式で定義されているパラメータを含んでもよい。
mimeType は type/subtype の組み合わせでメディアタイプおよびコンテナ形式を指定し、不明確な場合にはコードックやプロファイルのパラメータ（[RFC6381]）を指定します。コーデックによってはさらに追加のパラメータが必須または任意の場合があります。
audioBitsPerSecond, 型 unsigned long
オーディオトラックのエンコードに使用する合計目標ビット/秒値（存在する場合）。
videoBitsPerSecond, 型 unsigned long
ビデオトラックのエンコードに使用する合計目標ビット/秒値（存在する場合）。
bitsPerSecond, 型 unsigned long
すべてのビデオ・オーディオトラックの合計エンコード目標ビット/秒値。この値を指定すると、audioBitsPerSecond や videoBitsPerSecond より優先され、UAの裁量で各エンコーダに分配してもよい。
audioBitrateMode, 型 BitrateMode、デフォルト値 "variable"
オーディオトラックのエンコードに使用すべき BitrateMode を指定。
videoKeyFrameIntervalDuration, 型 DOMHighResTimeStamp
エンコードされたビデオストリームにおいてキーフレーム間の名目的な時間間隔（ミリ秒）。UAはこの値および videoKeyFrameIntervalCount を考慮してキーフレーム生成を制御。
videoKeyFrameIntervalCount, 型 unsigned long
エンコードされたビデオストリームでキーフレーム間のフレーム数。UAはこの値および videoKeyFrameIntervalDuration を考慮してキーフレーム生成を制御。

2.6. BitrateMode

enum BitrateMode {
  "constant",
  "variable"
};

2.6.1. 値

constant
固定ビットレートでエンコード。
variable
可変ビットレートでエンコードし、信号が複雑なときは多く、単純なときは少ないデータ量を使用できる。

2.7. RecordingState

enum RecordingState {
  "inactive",
  "recording",
  "paused"
};

2.7.1. 値

inactive
録画はされていない状態（開始前または停止後）。
recording
録画が開始され、UAがデータを取り込み中。
paused
録画中に一時停止され、まだ停止や再開が行われていない状態。

4. エラー処理

4.1. 一般原則

このセクションは規範的ではありません。

エラーが呼び出し時点で判定できる場合、UAは DOMException をスローします。 それ以外の場合は エラーイベントを発火する挙動となります。録画が開始され停止前にエラーが発生した場合、 blob を Blob のままエラー発生後UAは dataavailableイベントを発火し、その直後に stop イベントを発火します。 UAはプラットフォーム固有の最小/最大 Blob サイズや同時録画可能な MediaStreamTrack 数などの制限を設けてよいです。 この制限を超えた場合は致命的エラーになります。

4.2. エラーイベント

エラーイベントを発火するとは、ErrorEvent を eventConstructor として イベントを発火することです。

4.3. 例外サマリ

この文書で定義されている各例外は DOMException の特定タイプ（type）が使われます。

名前 説明
InvalidStateError 許可されていないオブジェクトや不適切なタイミングで操作が呼ばれた場合、または削除済み/除外済みソースオブジェクトに対し要求を行った場合。
NotSupportedError MIMEタイプ非対応またはMIMEタイプごとに録画できないトラック群などの理由で操作できなかった場合。追加情報は message 属性で可能な限りUAから提供。
SecurityError MediaStream の isolation properties のため MediaRecorder がアクセスできない場合。
InvalidModificationError 録画対象 MediaStream の MediaStreamTrack 構成が変更され、 継続録画できなくなった場合。

5. イベント概要

以下の追加イベントが MediaRecorder オブジェクトで発生します:

イベント名 インターフェース 発火条件
start Event UA が MediaStream からの録画を開始した時に発火。
stop Event UA が MediaStream からの録画を停止した時に発火。
dataavailable BlobEvent UA がアプリケーションにデータを戻すために発火。イベントの data 属性には録音データの Blob が入っている。
pause Event UA が MediaStream からの録画を一時停止した時に発火。
resume Event UA が MediaStream からの録画を再開した時に発火。
error ErrorEvent メモリ不足や録画中 stream にトラック追加・削除等の変更が起こり、継続録画できなくなった時などに発火。

6. プライバシーとセキュリティの考慮事項

このセクションは規範的ではありません。

MediaRecorder のデータソースは常に MediaStream となるので、セキュリティ上の多くは [GETUSERMEDIA] の「プライバシーとセキュリティ考慮事項」節に委ねられます。 特にソース MediaStream は 安全なコンテキストから供給されることが前提です。

6.1. リソース枯渇

ビデオおよびオーディオエンコーディングは多くの計算資源を消費します。 悪意のあるウェブサイトは、フレーム解像度やフレームレートを大きくすることでUAに過大な負荷をかけて動作不能にしようとする可能性があります。

MediaRecorder は start() 時に timeslice パラメータを使ってエンコード済みデータを一定期間保持できます。timesliceの値が大きすぎると大量データバッファリングを強いられ、動作のもたつきやメモリ枯渇を招きます。

UAは、エンコードやバッファリング過程でリソース枯渇を防ぐ対策を行うべきです。

6.2. 指紋採取

MediaRecorder はサポートされるビデオ・オーディオのMIME型を isTypeSupported() メソッドから知ることができます。また MediaRecorderOptions で明示しなかった場合、UAが最適と判断したコーデックや帯域幅割当てを選び、 type 属性経由で通知します（ondataavailableイベントで取得）。 MediaRecorderOptions の指定も尊重します。

悪意あるウェブサイトがこの情報を アクティブフィンガープリンティング に使う可能性があります。例：

  • UA機能差を元にデバイス／ハード特性・OSベンダ/バージョン差を推測: 例えば特定のコーデックやハードウェア支援エンコーダの利用可否や解像度/フレームレート上限など。

  • 性能計測（例:デフォルト帯域幅割当や異なる入力ベクトルでの負荷差計測）で推測。

UAは フィンガープリントの表面積を広げないため、可能な限り幅広いコーデック/MIME型をサポートし、 ハードウェアやOSに依存した差異を減らし、デバイス識別を困難にする対応を取る必要があります。 またUAの既定値は情報漏洩や過度な能力識別を防止する方向で設計すべきです。

7.2. ウェブカメラの動画と音声を録画する

この例は、MediaStream を使って映像＋音声の取得を getUserMedia() で行い、<video> タグに流し、それを録画して ondataavailable イベントで録画片を取得する例です。録画は MediaRecorder を stop() するか、録画された MediaStreamTrack が全て ended になるまで続きます。

以下の例も このCodepen でほぼ同じ形で確認できます。 
<html>
<body>
<video autoplay/>
<script>
  var recordedChunks = [];

  function gotMedia(stream) {
    // |video| は取得された MediaStream のライブ映像を表示します。
    var video = document.querySelector('video');
    video.src = URL.createObjectURL(stream);

    var recorder = null;
    try {
      recorder = new MediaRecorder(stream, {mimeType: "video/webm"});
    } catch (e) {
      console.error('Exception while creating MediaRecorder: ' + e);
      return;
    }

    recorder.ondataavailable = (event) => {
      console.log(' Recorded chunk of size ' + event.data.size + "B");
      recordedChunks.push(event.data);
    };

    recorder.start(100);
  }

  navigator.mediaDevices.getUserMedia({video: true, audio: true})
      .then(gotMedia)
      .catch(e => { console.error('getUserMedia() failed: ' + e); });
</script>
</body>
</html>
recordedChunks は例えば MediaRecorder Web Fundamentals記事の download() 関数などでファイル保存が可能です。
