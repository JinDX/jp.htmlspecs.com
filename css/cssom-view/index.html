概要

本仕様で導入されるAPIは、文書の表示ビューを調査・操作する方法を著者に提供します。これには、要素のレイアウトボックスの位置取得、スクリプトによるビューポートの幅取得、要素のスクロール操作などが含まれます。

CSSは、構造化された文書（HTMLやXMLなど）の描画を、画面・紙・音声などで記述するための言語です。

3.1. スクロール

ユーザーエージェントがスクロールを実行#perform-a-scroll参照箇所:3.1. スクロール4. Windowインターフェースの拡張6.1. Element Scrolling Members (2)する際、scrolling box box を、指定された位置 position、 関連する要素 element、およびオプションでスクロール動作 behavior（省略時は "auto"）に対して、 次の手順を実行します。

  1. 進行中の スムーススクロールを box で中断する。
  2. ユーザーエージェントが scroll-behavior プロパティを尊重し、次のいずれかが真の場合:
    • behavior が "auto" で、element が null でなく、かつその算出値 scroll-behavior プロパティが smooth である
    • behavior が smooth
    ...この場合、スムーススクロールで box を position までスクロールする。 それ以外の場合は 即時スクロールで box を position までスクロールする。

ユーザーエージェントが スムーススクロールを scrolling box box で position まで実行する場合、 ユーザーエージェント定義の方法で、ユーザーエージェント定義の時間かけてスクロール位置を更新する。スクロールが完了したとき、box のスクロール位置は position でなければならない。 スクロールはアルゴリズムまたはユーザーによって中断#smooth-scroll-aborted参照箇所:3.1. スクロールされる場合もある。

ユーザーエージェントが 即時スクロール#concept-instant-scroll参照箇所:3.1. スクロールを scrolling box box で position まで実行する場合、 box のスクロール位置を position に更新しなければならない。

文書の先頭へスクロールするには、文書 document に対し次を実行する:

  1. viewport を document に関連付けられた viewport とする。
  2. position を、viewport の scrolling area の 先頭エッジ を viewport の 先頭エッジ に合わせたときのスクロール位置とする。
  3. もし position が viewport の現在のスクロール位置と同じで、かつ viewport に進行中の スムーススクロール がなければ、これらの手順を中断する。
  4. スクロールを実行し、 viewport を position まで、関連要素として document のルート要素（あれば）、なければ null を使う。

注: このアルゴリズムは、HTMLで定義される #top フラグメント識別子にナビゲートしたときに使われます。[HTML]

    1. input を token とする。

    2. position を input の最初の文字に設定する。

    3. 空白をスキップする。

    4. 空白でも「=」でもない文字列を収集する。 name を収集した文字列のASCII小文字変換とする。

    6. 空白をスキップする。

このセクションはHTMLで定義されたイベントループと統合されます。[HTML]

resize手順を実行するよう要求されたとき、Document doc に対して、次の手順を実行する：

  1. doc を viewport の関連付けられた Document とする。

  2. doc が既に doc の 保留中のスクロールイベントターゲットに含まれていれば、これらの手順を中断する。

  2. 要素が既に doc の 保留中のスクロールイベントターゲットに含まれていれば、これらの手順を中断する。
